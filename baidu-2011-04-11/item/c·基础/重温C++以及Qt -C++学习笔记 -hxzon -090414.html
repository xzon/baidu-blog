重温C++以及Qt -C++学习笔记 -hxzon

1，C++风格的初始化，
T t(1,2); 相当于T t=new T(1,2);

2，构造函数冒号之后的是成员初始化列表，不能写入构造函数之中？？否则在调用之前会先调用默认父类构造函数？？

3，using namespace std; 使得我们可以把命名空间std中的所有符号都加入到我们程序的全局命名空间中。这样std::cin可以简写成cin。

4，int main(int argc,char* argv[] ) 第一个参数，是argv中字符数组的个数，argv数组中的每一项保存一个命令行字符串，第一个总是该可执行文件的名字，其余参数由命令行中的空格与制表符分割，如果要包含空格，需加双引号。

5，const在星号之后，表示修饰指针所指的变量，在星号之前，表示修饰指针。

6，类定义后有分号。静态成员在cpp中赋初值和定义，但是前不能再用static修饰，否则意义转变。会把变量的作用域由全局作用域变为文件作用域。

7，拷贝构造函数，和拷贝赋值运算。如果不定义，编译器会帮我们默认生成。
T::T(const T& other);
T& T::operator=(const T& other);

8，转换构造函数，当一个构造函数只有一个参数，并且参数类型不是本类型时。
例如Fraction frac(8);或者Fraction frac=8;
其实隐式调用了Fraction::Fraction(8);Fraction::operator=();
这也是编译器帮我们默认生成的转换赋值运算符。
T operator=(const T& x);

9，const成员函数不能改变自己所在对象的状态。const加在函数末尾。const也是函数重载。在cpp文件定义时，仍要有const修饰符。

10，子类方法覆盖父类方法，当父类的方法没有使用virtual修饰时，是没有多态性的。
析构函数必须为虚函数？？
纯虚函数 virtual QString getDimensions() =0; 表明不用给出定义。

二十三，继承详述
11，虚函数表指针，虚函数表
我们不建议在析构函数中调用虚方法，不会发生动态绑定。
如果析构函数是虚函数，那么编译器运行运行时才绑定。
如果一个类中定义了一个或多个虚方法，那么就应该定义一个虚析构函数，即使该析构函数为空。

12，解决多继承冲突，使用虚继承。子类一级继承时，加上virtual在父类前。这样，孙类一级继承两个相同父类的子类时，不会冗余爷爷的字段。Java使用接口，使得子类必须定义，避免了含糊。

13，public继承，意味着扩展，这是一种接口关系。private继承，相当于给派生类添加一个额外的部分充当private成员。这是一种实现关系。protected继承，像是给派生类添加了一个对象作为protected成员，只不过派生类可以访问该对象的this指针。

如果stack从vector公开派生，那么它会允许用户代码无限制的操作栈中所有元素，因为它含有vector的所有公共接口，如果是私有派生，那么用户代码就只能访问与栈定义一致的操作，vector接口被私有，内部使用。

如果stack继续派生xstack，它要使用vector接口，那么stack就应该改为保护继承，使得xstack可以使用vector接口。

14，管理容器，组合，聚合
管理容器是组合，当自己销毁时负责销毁自己的全部内容。
非管理容器是聚合，保留其他对象的索引。
qDeleteAll(container)。
QList重载了<<运算符，用于向列表中添加一项。

隐式共享类。qSort()。serializer模式。

15，QMap是值容器，因此不会管理堆对象，需自己管理内存。

}
