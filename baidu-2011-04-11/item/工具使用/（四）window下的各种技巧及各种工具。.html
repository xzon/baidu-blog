七、window中对等网络的安装
１、安装网卡及驱动程序
(1)打开主机箱,?将网卡插入相应的扩展槽,
(2)?对于pnp网卡,window会自动识别,如果有网卡的驱动程序,可以从软盘安装.
否则选择window自带的驱动程序
(3)?对于非pnp网卡则应事先了解window中闲置的IRQ号及I/O地址范围,?并使用
dos下的驱动程序进行设置,如果没有dos下的驱动程序,可通过跳线或对windows
中的其它设备的IRQ号及I/O地址进行改变,达到与其它设备没有冲突为原则.
２、配置各种协议
(1)选择"我的电脑"->"控制面板"->"网络",在"配置"项中点"添加";
(2)在要安装的网络组件中选择协议,点添加按钮;
(3)选择"microsoft"的"ipx/spc兼容协议"或"tcp/ip协议"
(4)在要安装的网络组件中选择客户,点添加按钮;
(5)在厂商栏中选择"microsoft",用户栏中选"microsoft网络用户"
(6)在要安装的网络组件中选择服务,点添加按钮;
(7)选择microsoft网络上的文件与达因机共享,按确定按钮
经过上述选择后,网络的配置中有"microsoft网络用户,网络适配器,ipx/spx或
tcp/ip协议,及microsoft网络上的文件与打印机共享四项.
(8)?在配置的文件与打印机共享机按钮,设置"允许其他用户访问我的文件"和允
许其他计算机使用我的打印机.
(9)?网络的标识中,设置计算机名,工作组及计算机说明,同组计算机的工作组名
必须相同.
(10)网络的访问控制中,设置共享资源访问控制方式为共享级用户访问控制.
单击完成,,插入window光盘,按照提示完成上述工作的安装.
(11)系统完成后,重新启动计算机,就可以在网上邻居中看到其它计算机.

十九、认识win95的启动文件与dos启动文件的关系
安装完win95后,dos下的三个文件io.sys,msdos.sys,command.com分别改名为
io.dos,msdos.dos,command.dos,win95的启动文件io.sys,msdos.sys,
command.sys,在多重启动中,扩展名改为w40.
(1)在以后的启动过程中,出现"Starting window 95..."的画面时,按下F4,进入
dos,那么dos下的三个文件改名为io.sys,msdos.sys,command.com.win95下的三
个文件改名为的io.w40(或winboot.sys),msdos.w40,command.w40,
以后进入dos的启动.
(2)当没按下F4时,?dos下的三个文件改名为io.dos,msdos.dos,command.dos,
win95下的三个文件又改回io.sys,msdos.sys,command.com,进入win95.
(3)相应的autoexec.bat在dos下的为autoexec.dos,win95下为autoexec.w40
相应的config.sys在 dos下的为config.doswin95下为config.w40
如果win95启动正常,dos不能启动,则应查找io.dos,msdos.dos,command.dos
文件是否正常,可以重系统启动盘中拷贝.
如果dos启动正常,win95不能启动,则应查找io.w40,msdos.w40,command.w40
文件是否正常,可以重系统启动盘中拷贝.
注意,在压缩盘中,这些文件在属主盘中,而不是在正常运行的盘中.



二十、还原win95注册表
win95注册表中,包含许多配置信息,不小心修改后会造成win95无法启动,这就需
要还原注册表.
(1)关闭系统,重启动并进入dos方式,进入windows子目录
(2)输入:
attrib -h -r -s system.dat
attrib -h -r -s system.da0
copy system.da0 system.dat
attrib +h +r +s system.dat
attrib -h -r -s user.dat
attrib -h -r -s user.da0
copy user.da0 user.dat
attrib +h +r +s user.dat
(3)重新启动机器,注册表就可以重新启动,须注意的是要经常备份上述二文件.

二十四、win95启动的奥秘
win95启动时需要的引导文件有:io.sys msdos.sys config.sys autoexec.bat
command.com
(1)io.sys 是一个不能更改的二进制文件,它包含了一些设置,config.sys中的
files 及buffers以及stacks的值要大于io.sys中的缺省值.
(2)msdos.sys是一个标准的文本文件,它是只读和隐含.系统的,使用attrib命令
将其属性修改后可以对其进行编辑.
1)path节的内容
1.1)windir=path(指定win95的路径)
1.2)winbootdir=path,指定启动win95所必须的文件所在的路径,缺省为win95安
装时所确定的路径.
1.3)hostwinbootdrv=directory,用于指定引导驱动器盘的根目录.
2)options节的内容
2.1)?bootdelay=m m为数值秒,?该数值表示在启动系统信息前,显示信息'start
windows 95'的延时秒数,为了保留足够的时间来按下功能键(F4,F8)选择启动,
可以增加m的数值.
2.2)bootfailsafe=1或0 是否在启动失败时采取安全模式启动
2.3)bootgui=1或0 启动win95时,是否自动进入图形用户界面(GUI)和整个操作
系统,为0是,启动后只进入dos7.0,而不引导其它信息.
2.4)bootkeys=1或0 是否采用功能键(F4 F8 shift)启动方式,在重新启动时,按
下shift键,可以不检测硬件设备而实现直接启动.
2.5)bootmenu=1或0 是否自动启动win95引导菜单,而不必按下F8,当它为1时,显
示win95启动时的引导菜单,这时可进行选择.
2.6)bootmenudefault=m m为win95引导菜单的缺省选项.它需要与bootmenu=1
相结合使用,并且bootmenudefault=m应该为数字值.
2.7)bootmenudelay=m,m为win95菜单缺省选项,在执行前延迟的秒数,它与
bootmenu=1相配合使用有效.
2.8)bootmulti=1 或0 是否引导以前的dos版本,当为1时,按下F4可以引导以前
的dos版本,缺省为1,若为1,则多出"perevious version of msdos"的选项.
2.9)bootwarn=1或0 是否在使用失败-安全菜单时报警,设为0,取消"安全模式"
启动后的警告信息及菜单.
2.10)bootwin=1或0 是否将win95作为操作系统引导,1为时,先出现win95画面后
又引导以前的dos版本菜单,最后进入dos提示符.
2.11)dblspace=1或0 是否加载dblspace.bin 1为加载
2.12)loadtop=1或0 是否把command.com drvspace.bin加载入高端内存
2.13)logo=1或0 是否在启动过程中显示win95的标志
2.14)network=1或0 是否加载网络部件,启动时若已经连上网络,则菜单多出"
safe mode with networking"选项.
2.15)doublebuffer=1或0 是否实现多中启动方式,为0时,启动后进入win95画面
后,又进入dos提示符,若想进入window 则输入"win",与boorgui类似.
3)关于autoexec.bat 与config.sys
实际上win95并不需要config.sys与autoexec.bat,只有需要实模式驱动程序或
更改io.sys的设置时才需要config.sys,若win95有不支持的驱动程序,也可通过
config.sys来加载.对于autoexec.bat也是一样.

二十六、win95启动菜单详解
1.normal
2.logged(\bootlog.txt)
3.safemode
4.safe mode with network support
5.step_by_step confirmation
6.comand prompt only
7.safe mode command prompt only
8.previous version of ms_dos.
1)normal 启动完整的windows 操作系统和GUI,等于没有按下F8,正常启动
2)logged(\bootlog.txt)选择此项除启动完整的windows和GUI,同时在根目录下
创建完整的引导标记文件(bootlog.txt).文件记录了win95启动的有关情况.
如果以后启动困难,可以使用它来查看是由那个软,硬件造成的.
3)safe mode 选择此项可以最小的加载启动win95,它仅仅启动必要的驱动程序,
显示模式采用标准VGA,如果系统由于驱动程序造成不正常,可以选择此项解决.
4)safe mode with network support
选择此项可以对在第三项启动最小win95的基础上,加载网络的实模式支持软件,
它允许访问网络.
5)step_by_step confirmation 选择此项可以对系统配置文件(config.sys)和
自动批处理(autoexec.bat)的每一项进行问答式执行,如果系统由于上述文件造
成启动不正常,可以选择此项.
6)command prompt only 选择此项可以启动windows的命令行提示符,但是没有
全部启动GUI和相应的命令,它几乎相当于启动一个DOS系统,可以运行基于DOS的
应用程序.?在GUI和相应的驱动程序出现问题时,使用它,以便进行文件维护和排
除故障.
7)safe mode command prompt only
选择此项可以最小方式启动windows,?是safe mode 和 command prompt only的
混合提,跳过系统配置文件和自动批处理文件,启动的是命令行提示符.实际上它
是在标准VGA显示模式下加载msdos.sys和command.com.在系统配置,批处理文件
以及GUI和相应的驱动程序同时出现时,?使用它可以全面进行文件维护和鼓掌检
测.
8)previous version of ms_dos(仅对于升级用户或安装了msdos的用户),他相
当于在引导以前的dos,与启动时按下F4相同.

二十九、声卡上的功能
1)line in和line out
line in是外部音频信号的输入接口,可将来自话筒,收录机及激光唱盘等声源的
声音采样,?存为数字声音文件,如将袖珍录音机的声源通过声卡接入电脑从而达
到边听磁带或广播边用电脑录音的功能.
line out是把声卡当做声源,接入音响的audio in接口,若音响效果号,则可听到
较号的效果.
2)声卡上的IDE接口
若电脑中配备双硬盘,对于光驱来说,就无法再加了这时我们可利用声卡上的IDE
接口接光驱,可以将声卡及光驱挑线适当改动即可.
如:ESS 16位声卡,jp2 open = enable 2nd ide close_disable 2nd ide open
就要将JP2跳线跳开.
同时光驱上master(ma)上应加一个跳线.
