JBuilder2008增加新特性 CodeGear王者归来？2008-04-03 来自：villa123

昨天CodeGear宣布最新JBuilder2008已经可以使用，该版本添加了新的代码复用工具以及来自于Instantiations的Swing设计器．也许大家对这些还不太熟悉，其实JBuilder是最古老的Java IDE之一，这次的发行版本似乎又带来了一些新的特性．JBuilder2008基于Eclipse的最新版本(Eclipse 3.3)，它已经囊括了一种被称为“ApplicationFactories (应用程序工厂)”特性。

ApplicationFactories鼓励开发人员复用，并且它使得开发人员无论从生产力还是开发速度上更上一层楼，尤其适合那些流动性大，相对分散的团队．只需要一个单一的简单工具，就可以让高级开发人员更专心的去交流，去获取式样书和相关建议．简而言之，ApplicationFactories已经将一个普通的IDE脱胎换骨，摇身一变成为应用程序专用IDE。

显然，通过ApplicationFactories你可以将你的代码和智慧存到repository的一个单独模块里去，分享给你的团队．也许你会说这算不了什么，但我知道，在IDE和Wiki(在这里指的是开发团队的知识库或文档之类的)来来回回的切换可让我们团队吃了不少苦，所以，我觉得这是一个非常有用的特性。

当然这个发布版本，来自Instantiations的Swing设计器也是亮点．它是Eclipse插件中心里使用率最高的插件之一，使得JBuilder的对Swing GUI的支持上升了一个台阶，与NetBeans的Matisse并驾齐驱。

JBuilder2008其它特性包括：

l 支持最新的主流商业和开源Java应用服务器
l 使用TeamInsight™ 和 ProjectAssist™协同团队开发
l 支持代码覆盖分析，支持内存和CPU性能分析
l 支持线程调试，添加request分析器
l UML建模和编码考古学(code archeology)
l Code metric和 Code audit功能(主要指代码静态检查部分)
