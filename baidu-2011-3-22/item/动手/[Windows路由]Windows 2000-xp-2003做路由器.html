Windows路由可以分为internet共享，就是我们一般用的方法，提供低质量的共享服务，这里提到的是路由与远程访问的一部分功能，比ICS强壮很多。
（1）首先打开服务“Routing and Remote Access”，并设置为自动，如下图

http://yueyuedeshijie.zq.sd.cn/tp/bbs/200608/29/2006829044142708.gif


http://yueyuedeshijie.zq.sd.cn/tp/bbs/200608/29/2006829046506411.gif


（2）打开命令行[运行CMD]，执行netsh，并执行如下图命令

http://yueyuedeshijie.zq.sd.cn/tp/bbs/200608/29/2006829050405116.gif


（3）重新启动计算机
启动后在运行netsh命令，如下图，假当你上网的连接叫“本地连接”，否则替换

http://yueyuedeshijie.zq.sd.cn/tp/bbs/200608/29/2006829053131582.gif



如果你有2块网卡，就把
add int 内部 private ==》add int 本地连接2 private
如果2块网卡
本地连接配置正常上网的IP
本地连接2配置一个私有地址，不要和上边的IP冲突就可以了，建议192.168.*.1 255.255.255.0,网关，DNS不配
客户机配置192.168.*.2-254(*=0-255),网关配192.168.*.1,DNS配本地连接的DNS（正常的就可以，不能配192.168.*.1,除非起了DNS服务，否则没有设置DNS代理，ICS可以这么配置）

如果只有一个网卡
本地连接配正常上网的IP
客户机和本地连接配一个网段的IP，网关配本地连接IP，DNS配能正常使用的DNS，别告诉我你的子网掩码是255.255.255.252,如果是卡去买个网卡吧，缺点就是容易造成IP冲突。
单网卡路由还可以使用VPN，WINDOWS天生支持PPTP（GRE）

因为起用了路由服务，本地网络里多了个[传入的连接]，通过配置传入的连接和客户端PPTP连接，就可以组建VPN网络。
