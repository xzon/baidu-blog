二。Java类成员初始化顺序

上述两种方式都可以进行成员初始化，那实际当我们实例化一个类的时候，到底编译器使用了哪一种方式，还是两种初始化都执行了，如果是，执行的顺序是怎样的，先看一段code：

/**
* @(#)chushihua.java
*
*
* @author hxzon
* @version 1.00 2008/5/11
*/


public class chushihua {
    private a a1;
    private a a2=new a(2);
    private a a3=new a(3);

    public chushihua() {
        System.out.println("chushihua");
        a1=new a(1);
        a3=new a(33);
    }

    public static void main(String[] args){
        chushihua c=new chushihua();
    }
}

class a{
    public a(int i){
        System.out.println(i);
    }
}

上述程序输出的结果是：

2     在构造函数之前就初始化了
3     在构造函数之前就初始化了
chushihua     构造函数调用
1     这时才初始化a1
33     重新初始化a3

我们看出，规定初始化总在实例化的时候执行了，而且不管构造函数有没有对成员进行初始化。可以认为，JAVA编译器保证任何成员变量在构造函数被调用之前已经被初始化，构造函数的初始化，只是对规定初始化的一个覆盖而已。同样对于静态成员变量也是一样的。
